# Contributing to AWS WAF Solver Comparison

We welcome contributions to this comparison repository! Your insights and updates on WAF solver tools are valuable.

## How to Contribute

1.  **Fork the Repository:** Start by forking the repository to your own GitHub account.
2.  **Create a Branch:** Create a new branch for your feature or fix (e.g., `feature/add-new-solver` or `fix/update-pricing`).
3.  **Make Changes:**
    *   **Tool Updates:** If you are updating a tool's details or pricing, modify the relevant section in `README.md`.
    *   **New Tools:** If you are adding a new tool, ensure it is a smaller, non-direct competitor to CapSolver, and update the comparison table and tool list in `README.md`.
4.  **Commit Changes:** Write clear, concise commit messages.
5.  **Submit a Pull Request (PR):**
    *   Ensure your branch is up-to-date with the main branch.
    *   Provide a clear description of your changes in the PR.
    *   Reference any relevant issues.

## Style Guide

*   **Language:** All content must be in English.
*   **Tone:** Maintain a professional and objective tone.
*   **Formatting:** Follow the existing Markdown and table formatting in `README.md`.
*   **Data:** Back up any claims about success rates or features with verifiable sources where possible.

Thank you for helping to keep this resource accurate and useful!
